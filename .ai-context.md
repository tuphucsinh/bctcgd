# Project Context: Báo cáo Tài chính Gia đình (BCTCGD)

## 1. Vision & Purpose (Mục tiêu)
- **Problem:** Quản lý thu chi, nợ, tiền lãi và phân bổ tài sản của cấu trúc gia đình bị phân mảnh, khó nhìn tổng quan.
- **User:** Gia đình (Tài khoản riêng biệt: Hiếu, Ly và Tài khoản chung Gia đình).
- **Core Value:** Giao diện Dashboard Bento Grid hiện đại (Dark Mode "Premium"), tích hợp nhập liệu tức thời, báo cáo trực quan cho phép chuyển user 1 click.
- **Các tính năng khác:** Theo dõi dư nợ gốc, chi phí lãi vay, phân bổ danh mục đầu tư (BĐS, Cổ phiếu, Tiền mặt).

## 2. Tech Stack (Công nghệ)
- **Frontend:** Next.js 16 (App Router), React 19, Tailwind CSS v4.
- **State Management:** React Hooks (`useState`, `useContext` sau này cho Auth).
- **Backend/API:** Supabase (PostgreSQL) - dùng `Transaction Pooler URI` cho local script và `REST/RPC` cho Frontend. Kèm theo Server Actions của Next.js để bảo mật.
- **Unique Libs:** 
  - `framer-motion`: Hỗ trợ Animation cao cấp.
  - `sonner`: Component hiển thị thông báo toast.
  - `recharts` (Dự kiến): Cung cấp biểu đồ trực quan.
  - `lucide-react`: Bộ icon giao diện.
  - `shadcn/ui` (Phiên bản dựa trên **Base UI/MUI**, KHÔNG PHẢI Radix UI).

## 3. Architecture & Data Flow (Kiến trúc & Luồng dữ liệu)
- **Pattern:** Folder-by-feature (App Router), tách biệt logic ở `src/lib/`, UI components tái sử dụng ở `src/components/ui/`.
- **Primary Data Flow:** Client Component (Dashboard/Modal) ➔ Gọi Next.js Server Actions (sắp làm) ➔ Bypass RLS (nếu cần bằng Service Role) hoặc dùng Auth Client ➔ Database Supabase (các schema `transactions`, `debts`, `assets`, `categories`).
- **Core Value:** Giao diện Dashboard Bento Grid hiện đại (Dark Mode "Premium"), thiết kế **Compact & Minimalist** (tối ưu diện tích), tích hợp nhập liệu tức thời, chuyển User nhanh bằng thanh **Toggle một chạm**.
- **Các tính năng khác:** Theo dõi dư nợ gốc, chi phí lãi vay, phân bổ danh mục đầu tư (BĐS, Chứng khoán, Tiền mặt).

## 4. Key Logic (Các Lỗi & Kinh nghiệm sinh tử - Đọc kỹ)
- **[CRITICAL] Component Base UI:** Thư viện giao diện đang cài đặt là **Base UI**. Khi dùng các component bọc (như `DialogTrigger`), **TUYỆT ĐỐI KHÔNG DÙNG** thuộc tính `asChild`. 
- **[NEW] Accessibility & Console Errors:** Phải truyền trực tiếp `children` vào thuộc tính `render` của `DialogTrigger` (ép kiểu `as React.ReactElement`) để tránh lỗi lồng thẻ button gây đỏ console và làm gián đoạn Next.js Error Overlay.
- **Compact UI Design:** Sử dụng bo góc `rounded-3xl` (24px) thay vì `2rem` để giao diện trông chuyên nghiệp và hiện đại hơn. Giảm padding/gap (`p-5`, `gap-4`) để hiển thị nhiều thông tin hơn trên một màn hình.
- **Deploy Vercel:** Do cấu trúc khác biệt, dự án bắt buộc cần file `vercel.json` định tuyến `outputDirectory: ".next"`.
- **Database Schema:** Đã khởi tạo hoàn tất qua file `runSql.js`. Tương tác trực tiếp bằng URI Pooler (ipv4) thay vì DB PGBouncer mặc định của Supabase.

## 5. Transfer List (File ưu tiên - Đọc theo thứ tự)
*Khi chuyển giao hoặc tiếp tục dự án, Agent nên đọc các file này trước để nắm 90% logic hệ thống:*
1. `src/app/page.tsx` (Logic tổng quan Dashboard Bento Grid, User Switcher).
2. `src/components/ui/transaction-modal.tsx` (Core logic nhập liệu, Pattern dùng Base UI chuẩn).
3. `supabase_schema.sql` (File schema gốc định nghĩa Database).
4. `src/lib/supabase.ts` (File khởi tạo Singleton của Supabase Client/Admin).
5. `vercel.json` (Trái tim của việc Deploy).

## 6. Team Rules & Conventions (Quy tắc)
- **Naming:** `camelCase` cho hàm/biến, `PascalCase` cho Component, `snake_case` cho database columns.
- **Styling:** Ưu tiên **UI Premium** (Bo góc lớn `2xl`, gradient borders tinh tế, shadow `shadow-2xl` hoặc glow shadow, glassmorphism với `backdrop-blur`).
- **Coding:** TypeScript strict mode, ES6+, Lambda, Shorthand logic. Phân tách rõ Client (`"use client"`) và Server components.
- **Git:** Commit message theo chuẩn Conventional Commits (`feat:`, `fix:`, `chore:`). Đẩy branch `main` lên repo `https://github.com/tuphucsinh/bctcgd` (Account: `tuphucsinh`).
- **Code Format:** Tối ưu hóa API Call. Cấm Yapping. **Chỉ cung cấp `diff`** hoặc đoạn code thay đổi. Không viết lại cả file lớn trừ phi tạo mới hoàn toàn.
- **Communication:** Ngôn ngữ Tiếng Việt, xưng hô Anh/Em. Phản hồi ngắn gọn, đi thẳng vào cốt lõi giải pháp.
- **Logic Check:** Luôn tự kiểm tra lỗi Typo và RLS policy trước khi đề xuất sửa logic Data Flow.
