# Project Context: Báo cáo Tài chính Gia đình (BCTCGD)

## 1. Vision & Purpose (Mục tiêu)
- **Problem:** Quản lý thu chi, nợ, tiền lãi và phân bổ tài sản của cấu trúc gia đình bị phân mảnh, khó nhìn tổng quan.
- **User:** Gia đình (Tài khoản riêng biệt: Hiếu, Ly và Tài khoản chung Gia đình).
- **Core Value:** Giao diện Dashboard Bento Grid hiện đại (Dark Mode "Premium"), tích hợp nhập liệu tức thời, báo cáo trực quan cho phép chuyển user 1 click.
- **Các tính năng khác:** Theo dõi dư nợ gốc, chi phí lãi vay, phân bổ danh mục đầu tư (BĐS, Cổ phiếu, Tiền mặt).

## 🛠️ Cấu trúc Database (v2) - CHI TIẾT
### 1. Phân loại Thu nhập & Chi phí (Categories)
- **Thu nhập (Income)**: 
  - *Chủ động*: Lương, Thưởng.
  - *Thụ động*: BĐS, Tài chính, Khác (Cần tính toán cho Gauge "Tự do tài chính").
- **Chi tiêu (Expense) - 10 mục**: 
  - Ăn uống, Di chuyển, Hóa đơn, Giao tế, Giải trí, Đồ dùng gia đình, Y tế, Học hành, Con cái, Khác.

### 2. Các loại Tài sản (Assets)
- **4 Loại**: Bất động sản, Tài chính, Vàng, Khác.
- **Trường dữ liệu**: Tên, Danh mục, Giá mua, Ghi chú.
- **Logic**: Checkbox "Trừ tiền mặt" (mặc định) khi thêm tài sản -> Tự động tạo giao dịch `ASSET_BUY`.

### 3. Quản lý Nợ & Cho vay (Debts)
- **Nợ (Borrow)**:
  - *Danh mục*: Nợ Ngân hàng, Cọc BĐS, Nợ khác.
  - *Phân loại*: Nợ ngắn hạn, Nợ dài hạn.
  - *Dữ liệu thêm*: Lãi suất (%), Trả gốc/Tháng, Tổng số tiền, Số tiền còn lại.
- **Cho vay (Lend)**:
  - *Dữ liệu*: Tên, Số tiền, Ngày trả (dự kiến), Ghi chú.

### 4. Giao dịch (Transactions)
- **Loại**: INCOME, EXPENSE, DEBT_PAYMENT, ASSET_BUY, ASSET_SELL.
- **Dữ liệu**: Số tiền, Ngày, Người chi (Hiếu/Ly/Joint), Danh mục, Ghi chú.
- **Liên kết**: `is_principal_payment` (để phân biệt trả gốc và lãi nợ).

## 3. Architecture & Data Flow (Kiến trúc & Luồng dữ liệu)
- **Pattern:** Folder-by-feature (App Router), tách biệt logic ở `src/lib/`, UI components tái sử dụng ở `src/components/ui/`.
- **Primary Data Flow:** Client Component (Dashboard/Modal) ➔ Gọi Next.js Server Actions (sắp làm) ➔ Bypass RLS (nếu cần bằng Service Role) hoặc dùng Auth Client ➔ Database Supabase (các schema `transactions`, `debts`, `assets`, `categories`).
- **Core Value:** Giao diện Dashboard Bento Grid hiện đại (Dark Mode "Premium"), thiết kế **VisionOS-inspired** (Premium Glassmorphism), tối ưu diện tích (Compact), chuyển User nhanh bằng thanh **Toggle một chạm**.
- **Animation Philosophy:** Hạn chế chuyển động thừa. Ưu tiên giao diện tĩnh (Static) để tăng độ sang trọng. Chỉ dùng hiệu ứng "thở" (breathing) cho các cảnh báo cực kỳ quan trọng (ví dụ: vượt ngưỡng ngân sách).
- **Tính năng mở rộng:** Theo dõi dư nợ gốc, chi phí lãi vay, phân bổ danh mục đầu tư (BĐS, Chứng khoán, Tiền mặt).

## 4. Key Logic (Các Lỗi & Kinh nghiệm sinh tử - Đọc kỹ)
- **[CRITICAL] Component Base UI:** Thư viện giao diện đang cài đặt là **Base UI**. Khi dùng các component bọc (như `DialogTrigger`), **TUYỆT ĐỐI KHÔNG DÙNG** thuộc tính `asChild`. 
- **[NEW] Accessibility & Console Errors:** Phải truyền trực tiếp `children` vào thuộc tính `render` của `DialogTrigger` (ép kiểu `as React.ReactElement`) để tránh lỗi lồng thẻ button gây đỏ console và làm gián đoạn Next.js Error Overlay.
- **Compact UI Design:** Sử dụng bo góc `rounded-3xl` (24px) cho các thẻ lớn và `rounded-full` cho các nút điều hướng/header để tạo sự mềm mại. Giảm padding/gap (`p-4` đến `p-5`) để hiển thị nhiều thông tin.
- **[NEW] Deficit Warning UI:** Khi chi tiêu vượt thu nhập, thẻ so sánh không dùng animation pulsing mà dùng **Static Glow Shadow** (cam nhạt, lan tỏa 40px) và viền cam để cảnh báo một cách tinh tế nhưng rõ ràng.
- **Header Synchronization:** Các bộ chọn chức năng (Tháng/Năm, User Switcher) phải đồng bộ tuyệt đối về `padding`, `font-size (text-xs)`, `font-weight (semibold)` và `ring-1 border` để tạo sự cân bằng thị giác.
- **Deploy Vercel:** Dự án bắt buộc cần file `vercel.json` định tuyến `outputDirectory: ".next"`.
- **Database Schema:** Đã khởi tạo hoàn tất qua file `runSql.js`. Tương tác trực tiếp bằng URI Pooler (ipv4) thay vì DB PGBouncer mặc định của Supabase.

## 6. Team Rules & Conventions (Quy tắc)
- **Naming:** `camelCase` cho hàm/biến, `PascalCase` cho Component, `snake_case` cho database columns.
- **Styling:** Ưu tiên **UI Premium** (Glassmorphism với `backdrop-blur-md`, gradient borders tinh tế `white/10` đến `white/20`, shadow `shadow-2xl` hoặc glow shadow đặc trưng cho trạng thái cảnh báo).
- **Navigation:** Mobile Bottom Navbar sử dụng nhãn tiếng Anh (e.g., **"Home"** thay vì "Tổng quan") và thiết kế **Ultra-compact** (giảm padding tối đa) để tối ưu không gian hiển thị.
- **Coding:** TypeScript strict mode, ES6+, Lambda, Shorthand logic. Phân tách rõ Client (`"use client"`) và Server components.
- **Git:** Commit message theo chuẩn Conventional Commits. Repo: `https://github.com/tuphucsinh/bctcgd`.
- **Code Format:** Chỉ cung cấp `diff`. Cấm Yapping.
- **Communication:** Ngôn ngữ Tiếng Việt, xưng hô Anh/Em. Phản hồi ngắn gọn.
- **Logic Check:** Luôn tự kiểm tra lỗi Typo và RLS policy trước khi đề xuất sửa logic Data Flow.
